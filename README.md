Цель проекта
Этот бот создан, чтобы помочь человеку быстро понять, что с ним может быть, и какие лекарства обычно назначают при той или иной болезни. Он не ставит диагноз и не заменяет врача, но даёт первичную информацию, на которую можно опереться до визита в поликлинику.
Главная идея в том, что многие люди едут к врачу или в аптеку просто потому, что не знают, с чем столкнулись. Если можно получить подсказку онлайн, часть поездок отпадает. Меньше поездок — меньше выхлопов от машин и автобусов. Это вклад в борьбу с глобальным потеплением.

Как бот помогает
Допустим, у вас поднялась температура или начался кашель. Вы пишете боту /symptoms температура, и он через пару секунд выдаёт список болезней, для которых характерен этот симптом. Это не готовый диагноз, а возможные варианты, о которых стоит знать.
Если врач уже поставил диагноз, например ангина, можно спросить /drugs ангина, и бот покажет список препаратов, которые обычно применяют при этой болезни. Эти данные берутся с официального медицинского справочника.

Функции бота
Две основные команды: /symptoms и /drugs.

Команда /symptoms
Вы пишете симптом (например, кашель, температура, насморк), и бот обращается к сайту health.mail.ru. Он берёт оттуда первые несколько статей, которые связаны с этим симптомом, и выдаёт их названия. Обычно это и есть возможные заболевания.
Работает быстро, без открытия браузера, через обычный HTTP-запрос. Если сайт недоступен, бот скажет об этом.

Команда /drugs
Вы пишете название болезни (ангина, бронхит, гипертония), и бот открывает справочник rlsnet.ru через Selenium в фоновом режиме. Он вводит ваш запрос в строку поиска, ждёт результатов и собирает список лекарств.
Этот способ используется потому, что rlsnet.ru не отдаёт данные без настоящего браузера. Но окно браузера не появляется — всё происходит в headless-режиме.

Особенности проекта
- Двухшаговый ввод: если вы забыли написать симптом или болезнь сразу после команды, бот попросит ввести их следующим сообщением. Это удобно на телефоне.
- При старте бот объясняет, как правильно вводить команды, чтобы не было ошибок.
- Никаких ссылок: результат приходит сразу в чат, не нужно никуда переходить.
- Устойчивость: для симптомов используется обычный парсинг HTML, который не боится блокировок. Для лекарств — Selenium с маскировкой, чтобы сайт не распознал автоматизацию.
- Код написан на Python и разделён на два файла: parsers.py (логика парсинга) и bot.py (сам бот). Это упрощает поддержку.

Установка
1. Установите зависимости: pip install -r requirements.txt
2. Вставьте токен бота в файл bot.py: TOKEN = "ваш_токен_от_BotFather"
3. Убедитесь, что установлен Google Chrome (нужен для /drugs)
4. Запустите: python bot.py

Заключение
Этот проект — пример того, как технологии могут делать жизнь проще и заодно помогать экологии. Даже такая мелочь, как отказ от лишней поездки в аптеку, уменьшает углеродный след. Если таких мелочей станет много, эффект будет заметным.
